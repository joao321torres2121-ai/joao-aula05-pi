## Criação do repositório
A criação do repositório é o primeiro passo para iniciar um projeto no GitHub. Ele funciona como uma pasta online onde ficam os arquivos do projeto e o histórico das alterações.

## Clone do repositório
O clone serve para baixar o repositório do GitHub para o computador, permitindo trabalhar nos arquivos localmente.

## Criação de branch
Uma branch é uma ramificação do projeto principal. Ela permite fazer alterações sem modificar o projeto principal.

## Registro de commits
O commit registra uma alteração feita no projeto. Ele salva no histórico o que foi modificado.

## Envio das alterações (push)
O push envia as alterações feitas no computador para o repositório no GitHub.

## Atualização do repositório local (pull)
O pull atualiza o repositório no computador com as alterações feitas no GitHub.

## Criação de Issues
As issues são usadas para registrar problemas, tarefas ou melhorias no projeto.

## Organização com Milestone
A milestone ajuda a organizar várias tarefas dentro de um objetivo ou etapa do projeto.

## Gerenciamento com Project
O Project permite organizar tarefas em formato de quadro, como um quadro Kanban.

## Abertura de Pull Request
O Pull Request é usado para revisar alterações antes de juntá-las com o projeto principal.

## Realização de Merge
O Merge é quando as alterações de uma branch são unidas com a branch principal.
